# Composition

Specs as composable, reusable contracts.

---

## 1. Purpose

This document explores how specs can reference other specs as dependencies, creating a library of verifiable capability modules. Atomic specs define single capabilities. Composite specs assemble them. Project specs apply those contracts to a concrete delivery context.
