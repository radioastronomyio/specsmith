# Iteration

Specs are living documents. The git history of a spec tells the story of how understanding deepened.

---

## 1. Purpose

This document explores spec evolution: how first-pass gates miss edge cases, how spec gaps surface as bugs rather than helpful agent improvisation, and how the iterative spec-build-validate cycle creates a virtuous learning loop. Covers the spec-as-code argument (versioned, diffable, reviewable in PRs) and the relationship between spec maturity and domain expertise.

---

*Content to be developed from validated practice. The Holdfast draw system miss and the milestone-over-milestone spec refinement are primary source material.*
